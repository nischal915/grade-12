Database Fundamentals

Data vs Information

Data refers to raw, unorganized facts that need to be processed. Data can be simple and unorganized until it is organized and given context.

Information is processed, organized, and structured data that provides context and enables decision making. For example, customer names and purchase amounts are data, but a sales report analyzing trends is information.

Features of Quality Information

  • Accuracy: Information should be error-free and precise
  • Completeness: All necessary data should be present
  • Consistency: Information shouldn't contradict other data
  • Timeliness: Information should be available when needed
  • Relevance: Information should be applicable to current needs
  • Accessibility: Information should be easily retrievable

Database and its Purpose

A database is an organized collection of structured information stored electronically in a computer system. Databases are designed to:

  • Store large amounts of data efficiently
  • Enable quick retrieval of specific information
  • Allow multiple users to access data simultaneously
  • Maintain data accuracy and consistency
  • Protect data through security measures
  • Provide backup and recovery capabilities

Database Terminologies

Table

A collection of related data organized in rows and columns. Tables are the fundamental structure in relational databases.

Field

A single piece of information in a table (column). Each field has a specific data type and constraints.

Record

A complete set of fields about one item (row). Also called a tuple in relational theory.

Keys

Special fields used to identify records and establish relationships between tables (Primary Key, Foreign Key, etc.)

Object

In object-oriented databases, an instance of a class that contains both data and methods.

Data Dictionary

A metadata repository that describes the structure, relationships, and meaning of data elements.

Database Management System (DBMS)

Introduction to DBMS

A DBMS is software that interacts with end users, applications, and the database itself to capture and analyze data. It serves as an interface between the database and end users or application programs, ensuring that data is consistently organized and remains easily accessible.

Objectives of DBMS

  • Provide efficient and convenient environment for data access
  • Eliminate data redundancy and inconsistency
  • Ensure data integrity and security
  • Support concurrent access with proper transaction management
  • Provide backup and recovery facilities
  • Maintain data independence between programs and data

Advantages of DBMS

Benefits

  • Reduced data redundancy
  • Improved data integrity
  • Enhanced data security
  • Efficient data access
  • Concurrent access control
  • Backup and recovery

Limitations

  • Complexity in design and implementation
  • High initial cost
  • Performance overhead
  • Vulnerability to failures
  • Requires skilled personnel

Types of Database Models

Hierarchical Model

Organizes data in a tree-like structure where each record has a single parent, except the root record. Uses parent-child relationships with one-to-many mapping between parent and child records.

Example: File systems, XML documents

Network Model

An extension of hierarchical model allowing multiple parents for a record. Uses graph structure with nodes representing records and edges representing relationships.

Example: CODASYL database systems

Relational Model

Stores data in tables (relations) with rows (tuples) and columns (attributes). Relationships between tables are established through foreign keys. Based on mathematical set theory and predicate logic.

Example: MySQL, Oracle, SQL Server

Entity-Relationship Model

A conceptual model that describes data as entities, attributes, and relationships. Used primarily for database design before implementation in a specific DBMS.

Components: Entities, Attributes, Relationships, Cardinality

Database Design Principles

Integrity Constraints

Domain Constraints

Restrict attribute values to a specific domain (data type, format, range).

Entity Integrity

Primary key cannot contain null values and must be unique.

Referential Integrity

Foreign key must match a primary key in the referenced table or be null.

Normalization

The process of organizing data to minimize redundancy by dividing tables and defining relationships.

1NF (First Normal Form)

  • Each table cell should contain atomic values
  • No repeating groups or arrays
  • Each record needs to be unique

2NF (Second Normal Form)

  • Must be in 1NF
  • No partial dependency (non-prime attributes dependent on proper subset of candidate key)

3NF (Third Normal Form)

  • Must be in 2NF
  • No transitive dependency (non-prime attributes dependent on other non-prime attributes)

Advantages

  • Reduces data redundancy
  • Improves data integrity
  • Simplifies database structure
  • Easier to maintain

Disadvantages

  • More complex queries with joins
  • Potential performance impact
  • Requires careful design

Centralized vs Distributed Databases

Centralized Database

All data stored at a single location

  • Advantages: Easier to manage, consistent, lower complexity
  • Disadvantages: Single point of failure, performance bottleneck

Distributed Database

Data spread across multiple locations

  • Advantages: Improved availability, better performance for local queries
  • Disadvantages: Complex management, consistency challenges

Database Security

Introduction to Database Security

Database security refers to the collective measures used to protect and secure database management systems from malicious threats and unauthorized access.

Security Challenges

  • Unauthorized access
  • SQL injection attacks
  • Data breaches
  • Denial of service
  • Weak authentication
  • Insufficient auditing

Security Measures

Authentication

Verify user identity through passwords, biometrics, etc.

Authorization

Define user privileges and access levels (RBAC)

Encryption

Protect data at rest and in transit

Auditing

Track database activities and changes

Backups

Regular data backups for recovery

Roles of Database Administrator (DBA)

  • Install and configure DBMS software
  • Design and implement database structures
  • Manage database security and user access
  • Monitor and optimize database performance
  • Implement backup and recovery plans
  • Troubleshoot database issues
  • Plan for capacity growth

Practical Database Topics

DDL and DML Languages

DDL (Data Definition Language)

Commands used to define and modify database structure:

  • CREATE - Create database objects
  • ALTER - Modify existing objects
  • DROP - Remove objects
  • TRUNCATE - Remove all records
  • RENAME - Rename objects

DML (Data Manipulation Language)

Commands used to manage data within objects:

  • SELECT - Retrieve data
  • INSERT - Add new records
  • UPDATE - Modify existing records
  • DELETE - Remove records
  • MERGE - UPSERT operation

SQL Data Types

String Types

  • CHAR(n) - Fixed-length string (n characters)
  • VARCHAR(n) - Variable-length string (max n characters)
  • TEXT - Variable-length large string
  • TINYTEXT - Very small string (255 chars)
  • LONGTEXT - Very large text (4GB)
  • ENUM - String with predefined values

Numeric Types

  • BIT - Binary value (0 or 1)
  • TINYINT - Very small integer (-128 to 127)
  • BOOLEAN - True/false values
  • INTEGER - Standard integer
  • FLOAT - Approximate numeric value
  • DOUBLE - Double precision floating point
  • DECIMAL(p,s) - Exact numeric with precision

Binary & Date Types

  • BINARY(n) - Fixed-length binary data
  • VARBINARY(n) - Variable-length binary
  • TINYBLOB - Small binary objects (255 bytes)
  • DATE - Date values (YYYY-MM-DD)
  • DATETIME - Date and time combination
