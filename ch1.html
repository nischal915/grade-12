Data & Information

Data: Raw, unorganized facts that need to be processed. Data can be something simple and seemingly random until it is organized.

Information: When data is processed, organized, structured or presented in a given context so as to make it useful, it is called information.

Features of Information

  • Accuracy: Information should be precise and free from errors
  • Completeness: All necessary details should be included
  • Timeliness: Information should be available when needed
  • Relevance: Information should be appropriate for the context
  • Consistency: Information should be consistent across different sources
  • Reliability: Information should come from trustworthy sources
  • Accessibility: Information should be easily accessible to authorized users

Database and its Purpose

Database: A structured collection of data that is stored and organized to allow efficient retrieval, insertion, and deletion of data.

Purpose of Database

  • To manage large amounts of data efficiently
  • To provide persistent storage for data
  • To eliminate data redundancy and inconsistency
  • To facilitate data sharing among multiple users
  • To maintain data integrity and security
  • To provide backup and recovery mechanisms
  • To support data independence between programs and data

Terminologies in Database

  • Table: A collection of related data held in a structured format within a database, consisting of columns and rows.
  • Field: A single piece of information in a table, represented by a column.
  • Record: A complete set of information about a single entity in a table, represented by a row.
  • Tuple: Another term for a row or record in a database table.
  • Object: In object-oriented databases, an instance of a class that contains both data and methods.
  • Keys: Special fields that establish relationships between tables and enforce integrity constraints.
    • Primary Key: Uniquely identifies each record in a table
    • Foreign Key: Establishes a link between two tables
    • Candidate Key: Any field that could serve as a primary key
    • Composite Key: A primary key consisting of multiple columns

Data Dictionary

Data Dictionary: A centralized repository of information about data such as meaning, relationships to other data, origin, usage, and format. It contains metadata (data about data).

Components of Data Dictionary

  • Data elements and their definitions
  • Data types and sizes
  • Relationships between data elements
  • Constraints and validation rules
  • Ownership information
  • Security and access permissions

Benefits of Data Dictionary

  • Improves data consistency and quality
  • Facilitates communication between stakeholders
  • Helps in database design and maintenance
  • Provides documentation for the database system
  • Assists in impact analysis when changes are made

Database Management System (DBMS)

DBMS: A software system that allows users to define, create, maintain, and control access to the database.

Objectives of DBMS

  • Data abstraction and independence
  • Efficient data access
  • Data integrity and security
  • Concurrent access and crash recovery
  • Reduced application development time

Advantages of DBMS

  • Controls data redundancy
  • Restricts unauthorized access
  • Provides persistent storage
  • Enforces integrity constraints
  • Provides backup and recovery

Disadvantages of DBMS

  • Complexity in design and implementation
  • High initial cost
  • Performance overhead
  • Vulnerability to failures
  • Need for specialized personnel

Types of Database Models

Hierarchical Model

Organizes data in a tree-like structure with parent-child relationships. Each parent can have multiple children, but each child has only one parent.

Network Model

An extension of the hierarchical model that allows each child to have multiple parents, creating a network of relationships.

Relational Model

Stores data in tables (relations) consisting of rows and columns. Relationships are maintained through common fields.

Entity-Relationship Model

A conceptual model that describes data as entities, attributes, and relationships. It's often used as a blueprint for database design.

Integrity Constraints and Types

Integrity Constraints: Rules that ensure the accuracy and consistency of data in a database.

Types of Integrity Constraints

Domain Constraints

Define the valid values for a given attribute. For example, age must be a positive integer.

Entity Integrity

Ensures that each row in a table is uniquely identifiable. The primary key cannot contain NULL values.

Referential Integrity

Ensures that relationships between tables remain consistent. A foreign key must match a primary key in another table or be NULL.

Key Constraints

Define which attributes can serve as keys to identify tuples uniquely.

Normalization

Normalization: The process of organizing data in a database to reduce redundancy and improve data integrity.

Normal Forms

First Normal Form (1NF)

Eliminates repeating groups by ensuring that each attribute contains only atomic values.

Second Normal Form (2NF)

Meets all requirements of 1NF and removes partial dependencies by ensuring that all non-key attributes are fully functionally dependent on the primary key.

Third Normal Form (3NF)

Meets all requirements of 2NF and removes transitive dependencies by ensuring that non-key attributes are not dependent on other non-key attributes.

Advantages of Normalization

  • Reduces data redundancy
  • Improves data integrity
  • Simplifies database design
  • Facilitates database maintenance

Disadvantages of Normalization

  • Increases complexity of queries
  • May reduce performance due to joins
  • Can make some operations more complicated

Centralized and Distributed Database

Centralized Database

A database that is located, stored, and maintained in a single location.

Advantages

  • Easier to manage and maintain
  • Lower hardware and software costs
  • Better data integrity and security

Disadvantages

  • Single point of failure
  • Performance bottlenecks
  • Limited scalability

Distributed Database

A database in which storage devices are not all attached to a common processing unit and are spread over multiple locations.

Advantages

  • Improved reliability and availability
  • Faster response for local queries
  • Easier expansion

Disadvantages

  • Increased complexity
  • Higher communication costs
  • Data integrity maintenance challenges

Database Security

Database Security: The protection of database against intentional or accidental threats that may be computer-based or non-computer-based.

Challenges in Database Security

  • Unauthorized access
  • SQL injection attacks
  • Data privacy concerns
  • Data integrity issues
  • Denial of service attacks

Security Measures

  • Authentication and authorization
  • Access control mechanisms
  • Encryption of sensitive data
  • Auditing and logging
  • Regular backups and recovery procedures
  • Firewalls and network security

DBA and Roles of DBA

Database Administrator (DBA): A person responsible for the design, implementation, maintenance, and repair of an organization's database.

Roles and Responsibilities of DBA

  • Database design and implementation
  • User management and security
  • Backup and recovery planning
  • Performance monitoring and tuning
  • Ensuring data integrity
  • Capacity planning
  • Troubleshooting and problem resolution
  • Documentation and reporting

ER Diagrams

ER Diagram: A visual representation of entities and their relationships to each other, typically used in database design.

Components of ER Diagrams

Entity

A real-world object or concept that can have data stored about it. Represented as rectangles.

Attributes

Properties or characteristics of an entity. Represented as ovals.

Relationships

Associations between entities. Represented as diamonds.

Cardinality

Defines the numerical relationship between two entities. Types include:

  • One-to-One (1:1)
  • One-to-Many (1:N)
  • Many-to-One (N:1)
  • Many-to-Many (M:N)

Degree

The number of entities participating in a relationship. Types include:

  • Unary (one entity)
  • Binary (two entities)
  • Ternary (three entities)

SQL (DDL and DML)

SQL (Structured Query Language): A standard language for storing, manipulating, and retrieving data in databases.

DDL (Data Definition Language)

Commands used to define the database structure:

  • CREATE - Creates database objects
  • ALTER - Modifies database objects
  • DROP - Deletes database objects
  • TRUNCATE - Removes all records from a table
  • RENAME - Renames database objects

DML (Data Manipulation Language)

Commands used to manipulate data:

  • SELECT - Retrieves data from database
  • INSERT - Adds new data to database
  • UPDATE - Modifies existing data
  • DELETE - Removes data from database

Data Constraints

  • NOT NULL - Ensures a column cannot have NULL value
  • UNIQUE - Ensures all values in a column are different
  • PRIMARY KEY - Uniquely identifies each row in a table
  • AUTO INCREMENT - Automatically generates a unique number when a new record is inserted

Operators

  • AND - Displays records where all conditions are true
  • OR - Displays records where at least one condition is true
  • NOT - Displays records where the condition is not true

WHERE Clause

Used to filter records based on specified conditions.

ORDER BY

Used to sort the result-set in ascending or descending order.

SQL JOIN

Used to combine rows from two or more tables based on a related column.

  • INNER JOIN: Returns records that have matching values in both tables
  • LEFT JOIN: Returns all records from the left table, and matched records from the right table
  • RIGHT JOIN: Returns all records from the right table, and matched records from the left table
  • FULL JOIN: Returns all records when there is a match in either left or right table

Database Management System (DBMS) - Detailed Q&A

1. What is Database and DBMS? List out the advantages and disadvantages of DBMS.

A Database is a structured collection of data that can be easily accessed, managed, and updated. It helps in organizing data efficiently and allows users to perform operations like retrieval, insertion, and deletion.

A DBMS (Database Management System) is a software system that enables the creation, maintenance, and use of databases. It acts as an interface between the database and the end users or application programs.

Advantages of DBMS:

  • Minimizes data redundancy by integrating all data in a single database.
  • Ensures data consistency and integrity across applications.
  • Improves data security with access controls and user authentication.
  • Supports data backup and recovery mechanisms.
  • Allows concurrent access and efficient data manipulation.

Disadvantages of DBMS:

  • High initial and maintenance cost.
  • Requires trained personnel to manage the system.
  • Hardware and software complexity.
  • Performance overhead in handling large data under high traffic.

3. Explain the different models of DBMS with advantages and disadvantages.

Hierarchical Model:

Organizes data in a tree-like structure with parent-child relationships.

Advantage: Fast data access for hierarchical data.

Disadvantage: Difficult to manage complex relationships.

Network Model:

Data is organized using a graph, allowing multiple parent-child relationships.

Advantage: Flexible relationship management.

Disadvantage: Complex structure and navigation.

Relational Model:

Data is stored in tables with rows and columns.

Advantage: Easy to use and widely supported.

Disadvantage: Performance may be slower with very large data.

4. What is a relational database? How is it different from other database models?

A relational database is a type of database that stores data in tabular form. Each table (relation) contains rows (records) and columns (attributes). It uses structured query language (SQL) for data operations.

Differences:

  • Based on mathematical relations unlike hierarchical/network models.
  • More flexible and user-friendly due to SQL.
  • Supports constraints, normalization, and better data integrity.

5. What is data redundancy? How does DBMS help in reducing it?

Data redundancy refers to unnecessary repetition of data in different locations. It leads to wasted storage and potential inconsistency.

DBMS minimizes redundancy by storing data centrally and using keys to relate tables. This avoids duplication and improves integrity and efficiency.

7. Who is Database Administrator (DBA)? What are the major responsibilities of DBA?

A Database Administrator (DBA) is a person responsible for managing and maintaining database systems.

Responsibilities include:

  • Database design and implementation.
  • Monitoring performance and optimizing queries.
  • Ensuring data security and access control.
  • Creating backup and recovery strategies.
  • Managing users and roles.

8. Define normalization. Explain 1NF, 2NF and 3NF with suitable examples.

Normalization is a technique to design relational databases to reduce redundancy and improve data integrity.

1NF (First Normal Form):

All attributes should have atomic values.

Example:

Invalid: {Name: "John", Phone: "123,456"}
Valid: {Name: "John", Phone: "123"} and another row with "456"

2NF (Second Normal Form):

No partial dependency of non-key attributes on part of a composite key.

3NF (Third Normal Form):

No transitive dependency. Non-key attributes should not depend on other non-key attributes.

9. Explain the terms: primary key, foreign key, and candidate key with examples.

Primary Key:

Uniquely identifies each record in a table.

Example: StudentID in Students table.

Foreign Key:

Refers to primary key of another table to establish relationship.

Example: CourseID in Enrollment table refers to Courses table.

Candidate Key:

All possible keys that can act as primary key. One of them is chosen as primary key.

10. What is SQL? Explain its components and common functions.

SQL (Structured Query Language) is a standard language to store, manipulate, and retrieve data from relational databases.

Components:

  • DDL (Data Definition Language): CREATE, ALTER, DROP
  • DML (Data Manipulation Language): SELECT, INSERT, UPDATE, DELETE
  • DCL (Data Control Language): GRANT, REVOKE
  • TCL (Transaction Control Language): COMMIT, ROLLBACK

11. Write SQL commands to perform the following operations:

a. Create a table named students with the fields: Id, Name, Class, and Marks.

b. Insert records into the students table with appropriate values.

c. Display all records from the students table.

d. Update the marks of a student whose name is 'Anita' to 85.

e. Delete the record of the student with id = 3.

a. Create table:

CREATE TABLE students (
    Id INT PRIMARY KEY,
    Name VARCHAR(50),
    Class VARCHAR(20),
    Marks INT
);

b. Insert records:

INSERT INTO students (Id, Name, Class, Marks) VALUES 
(1, 'Anita', '10A', 78),
(2, 'Rahul', '10B', 88),
(3, 'Neha', '10A', 92);

c. Display all records:

SELECT * FROM students;

d. Update marks:

UPDATE students
SET Marks = 85
WHERE Name = 'Anita';

e. Delete record:

DELETE FROM students
WHERE Id = 3;
